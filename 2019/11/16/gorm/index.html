gorm

发表于 2019-11-16 | 分类于 技术总结

gorm是golang中的一种ORM技术，因为由国人开发，因此在国内市场中使用较多，且文档也相对丰富。在开发过程中主要参考社区贡献的手册：gorm中文手册

gorm

虽然社区的中文手册已经很完美了，但是在实际使用过程中发现，正是由于其过于完美，往往会找不到重点，因此这里主要记录一下使用过程中踩到的各种坑，已经对应的方法。

关系

gorm中的关系有：

  1. belong to
  2. has one
  3. has many
  4. many2many

belong to和has one之间，没有严格的限制，更像是语意上的理解，例如：User has one credit, user belong to one company。但是gorm中对这两种关系做了区分，区分方式是：模型包含关系。

belong to

User属于Company，因此User必要外键到对应的Company上，但是为了表示这种belong to的关系，User还必要包含Company的模型。

has one

User 拥有Credit，因此User中有Credit字段，同时Credit属于User，因此必然会外键到对应的User中。
