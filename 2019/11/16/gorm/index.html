gorm

发表于 2019-11-16 | 更新于: 2019-11-21 | 分类于 技术总结

gorm是golang中的一种ORM技术，因为由国人开发，因此在国内市场中使用较多，且文档也相对丰富。在开发过程中主要参考社区贡献的手册：gorm中文手册

gorm

虽然社区的中文手册已经很完美了，但是在实际使用过程中发现，正是由于其过于完美，往往会找不到重点，因此这里主要记录一下使用过程中踩到的各种坑，以及相关的感悟。以下基于v1.9.11讨论。

表关系

gorm中的关系有：

  1. belong to
  2. has one
  3. has many
  4. many2many

belong to和has one之间，没有严格的限制，更像是语意上的理解，例如：User has one credit, user belong to one company。但是gorm中对这两种关系做了区分，区分方式是：模型包含关系。

belong to

User属于Company，因此User必要外键到对应的Company上，但是为了表示这种belong to的关系，User还必要包含Company的模型。这里可以看到，其实Company并不会在产生真实的sql语句。

has one

User 拥有Credit，因此User中有Credit字段，同时Credit属于User，因此必然会外键到对应的User中。在真实的sql语句中，只创建了credits表，并没有对users表做任何对修改。

基本操作

对数据库表的操作主要有增删改查，其中对于没有关联关系的表的删除比较简单，直接通过gorm的create，update等进行操作即可；但如果表格之间有关联关系，则需要通过gorm的Association来进行处理。

社区文档中对于增删查改的描述，个人觉得比较分散，这里添上自己的理解，重新组织一次，这里主要讲解create，其他几种基本操作与此类似。
